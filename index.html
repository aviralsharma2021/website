Etihad Airways · 2025 – Present
Solo-built

AI code attribution pipeline — origin of the AI productivity initiative

There was an organisational KPI to increase AI adoption in engineering — but no way to measure it. I identified that gap, defined what "AI-written code" means operationally, and built this pipeline solo. The baseline it produced is what justified and measured the multi-agent framework investment that followed. It now runs bi-weekly in Azure DevOps across every codebase in the enterprise data management team.

  • Six-bucket attribution taxonomy — All Human → Human Led → Inconclusive → Machine Assisted → Machine Led → All Machine — rather than a binary label; surfaces ambiguity honestly instead of forcing a coin-flip
  • Deterministic LLM classification using a transparent signal-weighting formula — each identified code signal carries a declared weight (high / medium / low), accumulating from a neutral 0.5 prior; every classification decision is fully auditable because it feeds an organisational KPI
  • Borderline resampling with multi-seed calls (seeds 11, 23, 37) for uncertain middle-zone chunks; confidence penalised by spread across seeds so disagreement is surfaced, not hidden
  • Per-file XAI audit log (JSONL) recording every signal, weight, and LLM call — if a stakeholder challenges a classification, the exact reasoning chain is inspectable
  • Parallel execution — 16 file workers × 20 concurrent LLM calls — with rate-limit detection, exponential backoff, and automatic fallback to sequential to prevent pipeline failure
  • Idempotent Unity Catalog export (DELETE-then-INSERT, 400-row batching) across four granularity levels: files → directories → applications → projects

Etihad Airways · 2025 – Present
Team contribution

Multi-agent data engineering assistant framework

Governance-first multi-agent framework for Databricks Lakehouse engineering. Contributed to agent boundary design, context injection strategy, and a four-tier TDD framework that addresses the endemic problem of over-relying on mocked unit tests and never validating real system behaviour end-to-end.

  • Playwright Tier C smoke testing — Playwright MCP drives a real browser session against the live FastAPI Swagger UI; catches middleware ordering, DI wiring, CORS config, streaming chunking, and schema mismatches that mocked HTTP clients structurally cannot surface
  • Golden fixture TDD loop — conversation fixtures store behaviour constraints (expected route, tool selection, response intent) not exact strings, making tests deterministically runnable against probabilistic LLM outputs
  • Four tiers — Tier A: deterministic unit/integration (mandatory CI, no real LLM); Tier B: prompt regression, flag-gated; Tier C: Playwright E2E against live endpoints; Tier D: DeepEval/RAGAS batch evaluation logged to MLflow with threshold gates
  • Approval-gate orchestration — Planner generates a full plan, halts for human approval, then delegates; human decision treated as a first-class architectural primitive, not an error handler — rare in agentic systems
  • Five-layer context injection — global governance → agent role → skill procedures → live repo reads → MCP calls (Atlassian, Azure DevOps, Databricks); agents arrive with full context without user prompting
